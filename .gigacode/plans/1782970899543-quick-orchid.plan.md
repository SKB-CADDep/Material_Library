# План декомпозиции проекта Material_Library (Обновлённая версия с PyQt6)

## 1. Схема декомпозиции

### Текущие требования:
- **UI фреймворк**: **PyQt6** (переход с tkinter)
- **Структура кода**: Модульная (src/core/, src/services/, src/ui/)
- **Конфигурация**: JSON файлы для всех констант
- **Документация**: Markdown в каталоге docs/
- **Тесты**: Unit тесты в каталоге tests/

### Задача:
**Создать новую структуру каталогов, перенести существующий код по модулям, вынести константы в JSON, переписать UI на PyQt6**

## 7. Labor Intensity (с переходом на PyQt6)

**Реальная оценка: 40-48 часов** (~5-6 рабочих дней) для одного опытного разработчика (Senior Python Developer)

| Этап | Время | Описание |
|------|-------|----------|
| Подготовка инструкции | 2 часа | Конвертация TXT → MD (instruction_list.txt, change_list.txt) |
| Подготовка структуры | 4 часа | Создание каталогов, pyproject.toml, requirements.txt с PyQt6 |
| JSON конфиги | 8 часов | Вынос констант в JSON (7 файлов) + валидация |
| Миграция модулей | 8 часов | Создание базовых сервисов (unit_manager, source_manager, material, audit_service) |
| Перенос кода + PyQt6 | 16-24 часа | Перенос main.py в src/ui/ + переписка tkinter → PyQt6 |
| Интеграция | 4 часа | Подключение импортов, проверка работоспособности |
| Тестирование | 4 часа | Unit тесты, проверка функциональности |
| **Итого** | **40-48 часов** | **~5-6 рабочих дней** |

**Дополнительные 8-16 часов** за счет переписки tkinter → PyQt6 для всех виджетов:
- `tk.Tk()` → `QApplication(sys.argv)`
- `ttk.Frame` → `QWidget`
- `ttk.Notebook` → `QTabWidget`
- `ttk.Treeview` → `QTableView` + `QStandardItemModel`
- `ttk.Combobox` → `QComboBox`
- `FigureCanvasTkAgg` → `FigureCanvasQTAgg`
- и т.д.

## 12. Финальная структура

После завершения декомпозиции:
- `main.py` - точка входа, загружает `src/ui/main_app.py` (PyQt6 QApplication)
- `src/` - модульная структура кода
- `config/` - JSON конфигурационные файлы
- `resources/` - текстовые файлы, инструкции
- `tests/` - unit тесты
- `docs/` - документация в Markdown

---

**Примечание (с PyQt6)**: После завершения декомпозиции все константы будут вынесены в JSON файлы, код будет организован по модулям, документация в Markdown, UI переписан на PyQt6.

**Примечание (без PyQt6)**: После завершения декомпозиции все константы будут вынесены в JSON файлы, код будет организован по модулям, документация в Markdown, но UI останется на tkinter.
