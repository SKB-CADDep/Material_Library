# Распределение задач Material Library

Оценка для разработчика, знакомого с проектом (`main.py`, JSON-схема материалов).  
Включает разработку, самопроверку и правки по итогам ревью. Не включает время заказчика на ответы.

Связанный документ: `[tasks.md](tasks.md)` — формулировки, приоритеты, зависимости.

**Занятость:** 40 ч/нед на каждого · **Ёмкость команды:** 80 ч/нед


| Фаза                     | Задачи | Часы       | Календарь  |
| ------------------------ | ------ | ---------- | ---------- |
| **1** — архитектура      | 18–28  | ~236 ч     | ~3 нед     |
| **2** — функциональность | 1–17   | ~240 ч     | ~3 нед     |
| **Итого**                | 1–28   | **~476 ч** | **~6 нед** |


**Баланс:** A ≈ **236 ч** · B ≈ **240 ч** (разница ≤ 4 ч)

---

## Принцип распределения

1. **Параллельные треки** — каждый разработчик ведёт свой стек с минимумом блокировок.
2. **Владение модулем** — все задачи одной вкладки / слоя у одного человека.
3. **Стыки через контракт** — не через ожидание чужого кода; 4 короткие встречи за проект (см. ниже).


| Трек                    | Человек | Фаза 1                                             | Фаза 2                                  |
| ----------------------- | ------- | -------------------------------------------------- | --------------------------------------- |
| **Данные и сервисы**    | **B**   | Схема, каталог свойств, миграция, PyQt-логика      | Редактор материала, источники в таблице |
| **UI и инфраструктура** | **A**   | Справочники, units, PyQt-оболочка, сборка, CRUD UI | Подбор материала, tooltips, расчёты     |
